# Config Overrides

Use `--config` with YAML to provide business context.

- Profiling fields (row/null/distinct counts) remain observed evidence.
- Config can override description/display name/semantic metadata and add review notes/caveats/business rules.
- Provenance fields in `data_dictionary.json`:
  - `description_source`
  - `display_name_source`
  - `semantic_role_source`
